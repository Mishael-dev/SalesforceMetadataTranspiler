# Field Documentation: Multiselect Picklist

This document breaks down the Salesforce **Multiselect Picklist Custom Field XML** and separates the metadata into categories based on how the transpiler, LLM, and UI will interact with them.

---

Stored in `picklist_multi_select__c.field-meta.xml`.

## 1. The Outer Container

This wraps the field definition in Salesforce Source Format (SFDX).

```xml
<?xml version="1.0" encoding="UTF-8"?>
<CustomField xmlns="http://soap.sforce.com/2006/04/metadata">
    <!-- Field content goes here -->
</CustomField>
```

## 2. Core Field Configuration (LLM Generated)

These are the primary intent fields. The LLM must decide these to define the business logic and identity of the picklist field.

```xml
<fullName>picklist_multi_select__c</fullName>
<label>picklist_multi_select</label>
<description>picklist description</description>
<inlineHelpText>picklist help text</inlineHelpText>
<required>false</required>
<valueSet>
    <restricted>true</restricted>
    <valueSetDefinition>
        <sorted>false</sorted>
        <value>
            <fullName>value 1</fullName>
            <default>false</default>
            <label>value 1</label>
        </value>
        <value>
            <fullName>value 2</fullName>
            <default>false</default>
            <label>value 2</label>
        </value>
        <value>
            <fullName>value 3</fullName>
            <default>false</default>
            <label>value 3</label>
        </value>
    </valueSetDefinition>
</valueSet>
```

## 4. Others (Transpiler Injected)

These fields are considered platform-required metadata and are automatically added by the transpiler. They ensure the XML is valid for the Salesforce Metadata API but do not require input from the LLM or the end-user.

## 5. Validation Rules

This section documents the validation rules applied to **Multiselect Picklist** fields at each validation layer in the transpiler pipeline.

### Level 1: Structural Validation (The Shape)

Ensures the JSON matches the Multiselect Picklist field schema.

* `type` must be exactly `MultiselectPicklist`.
* `label` must be a non-empty string.
* `fullName` must be a valid Salesforce custom field API name ending in `__c`.
* `required` must be a boolean.
* `valueSet.values` must be an array of objects containing `fullName`, `label`, and `default`.
* `valueSet.restricted` and `valueSet.sorted` must be booleans.

---

### Level 2: Logical Validation (The Context)

Ensures internal consistency and field dependencies.

* `valueSet.restricted` set to `true` enforces that only defined picklist values can be selected.
* `valueSet.sorted` determines whether the picklist options are displayed in sorted order.
* Each `value.default` can only be `true` for at most one value.
* `required` controls whether at least one selection is needed.

---

### Level 3: Platform Constraints (The Linter)

Enforces hard Salesforce platform rules that are not reliably expressible in Zod schemas and must be checked via custom logic.

* `fullName` (API name, excluding `__c`) must not exceed **40 characters**.
* `label` must not exceed **40 characters**.
* Maximum of **1,000 picklist values**.
* Multiselect picklist cannot be unique or an external ID.
* Value labels must be unique and cannot exceed **40 characters**.
* No more than **255 characters** per individual picklist value.

---

### Level 4: State & Conflict Validation (The Pre-Flight)

An optional runtime validation layer that inspects the target Salesforce org before deployment.

* Verify that no existing field with the same `fullName` already exists on the target object.
* Prevent changes to `type` (MultiselectPicklist â†’ other types is illegal).
* Prevent changes to picklist values if records already exist using those values.
* Validate that the target object is **not managed-package locked**.
* Detect namespace collisions in namespaced orgs.

Failures at this level should be surfaced as **pre-deployment conflicts**, not schema errors.
