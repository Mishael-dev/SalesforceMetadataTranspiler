# Object Documentation

This document breaks down the Salesforce Kitchen Sink Custom Object XML and separates **core object configuration fields** from **other optional or metadata elements**.

---

Stored in `Kitchen_Sink__c.object-meta.xml`

## 1. The outer container

This wraps the fields.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<CustomObject xmlns="http://soap.sforce.com/2006/04/metadata">

</CustomObject>
```

## 1. Core Object Configuration Fields

These are the primary fields that define the identity and behavior of the custom object.

```xml
<label>Kitchen Sink</label>
<pluralLabel>Kitchen Sinks</pluralLabel>
<description>This is a full-featured kitchen sink object including all fields and settings</description>
<nameField>
    <label>Kitchen Sink Name</label>
    <trackHistory>false</trackHistory>
    <type>Text | Auto Number</type>
</nameField>
```

## 2. Supporting Features

These fields will not be generated by the llm but they will be displayed on the react flow interface incease the user wants to edit them. 
```xml
<deploymentStatus>Deployed</deploymentStatus>
<allowInChatterGroups>true</allowInChatterGroups>
<enableActivities>true</enableActivities>
<enableBulkApi>true</enableBulkApi>
<enableFeeds>false</enableFeeds>
<enableHistory>true</enableHistory>
<enableLicensing>false</enableLicensing>
<enableReports>true</enableReports>
<enableSearch>true</enableSearch>
<enableSharing>true</enableSharing>
<enableStreamingApi>true</enableStreamingApi>
```
